前段时间想找个适合走 IPv6 的国外 VPS/云服务区域，手里 AWS CLI 已经配好了，就顺手用 Lightsail 做了次实测。记录一下过程和结论，方便以后参考。

## 测试思路

直接用 Lightsail API 的 IPv6 端点测意义不大，因为那些端点本身没有 AAAA 记录。要真实反映 "本机 IPv6 到 Lightsail 实例" 的质量，只能在不同区域临时创建实例，然后从本机跑 iperf3 测试。

选了 5 个有代表性的区域：

- `ap-southeast-1`（新加坡）
- `ap-northeast-1`（东京）
- `us-west-2`（俄勒冈）
- `eu-west-1`（爱尔兰）
- `us-east-1`（弗吉尼亚）

每个区域开一台最便宜的 Lightsail Nano（Amazon Linux 2023），安装 iperf3 服务端，打开 5201 端口，测完立刻删掉。

## 测试方法

- **延迟**：ICMP 被本地网络/运营商阻断，改用 TCP 握手延迟，连续 10 次连接 iperf3 的 5201 端口取平均。
- **带宽**：iperf3 分别测试 "本机 → 实例"（上传）和 "实例 → 本机"（下载）两个方向。
- **IPv4 对比**：顺手用 IPv4 也跑了一组下载，作为参照。
- **稳定性**：对新加坡和东京两个重点区域各复测了 3 轮。

## 结果

| 区域 | 地点 | TCP 延迟 avg / 抖动 | IPv6 上传 | IPv6 下载 | IPv4 下载 |
|------|------|---------------------|-----------|-----------|-----------|
| `ap-southeast-1` | 新加坡 | **82.9 ms / 2.33 ms** | **48.6 Mbps** | **246.8 Mbps** | 217.7 Mbps |
| `ap-northeast-1` | 东京 | 126.6 ms / 4.82 ms | 31.8 Mbps | **0.4 Mbps** ⚠️ | 251.5 Mbps |
| `us-west-2` | 俄勒冈 | 199.1 ms / 18.92 ms | 13.6 Mbps | 78.5 Mbps | 82.3 Mbps |
| `eu-west-1` | 爱尔兰 | 225.2 ms / 44.02 ms | 2.9 Mbps | 4.0 Mbps | 1.0 Mbps |
| `us-east-1` | 弗吉尼亚 | 289.4 ms / 42.47 ms | 1.9 Mbps | 50.7 Mbps | 30.1 Mbps |

复测结果：

- **新加坡**：下载 239 / 251 / 250 Mbps，上传 51 / 40 / 43 Mbps，非常稳。
- **东京**：下载 1.3 / 0.6 / 0.8 Mbps，上传 2.5 / 13.2 / 9.6 Mbps，抖动很大。

## 结论

如果目标就是 IPv6，**首选 `ap-southeast-1`（新加坡）**：延迟最低、带宽最高、抖动最小。东京虽然 IPv4 很强，但 IPv6 下载基本不可用，暂时排除。俄勒冈可以作为北美备选。

当然这只是一个时间点的测试结果，受本机网络、运营商 IPv6 路由和 AWS 实时网络状况影响，后续再用的话建议再简单验证一下。
